# Entities & Slots — Πλήρης Οδηγός

## Επισκόπηση

Στο Rasa, τα **entities** και τα **slots** είναι δύο ξεχωριστές αλλά στενά
συνδεδεμένες έννοιες:

- **Entity** = Μια πληροφορία που **εξάγεται** από το μήνυμα του χρήστη.
  Είναι θέμα **NLU** — ανίχνευση κατά την ανάλυση κειμένου.
- **Slot** = Μια μεταβλητή που **αποθηκεύεται** στη μνήμη της συνομιλίας.
  Είναι θέμα **Dialogue** — επηρεάζει τη ροή της συζήτησης.

```
  ┌──────────────────────────────────────────────────────────┐
  │                    ENTITY vs SLOT                        │
  │                                                          │
  │   Entity = "Τι βρήκα στο μήνυμα;"  (ανίχνευση)           │
  │   Slot   = "Τι θυμάμαι;"           (αποθήκευση)          │
  │                                                          │
  │   Entity → εξάγεται → αποθηκεύεται σε Slot → επηρεάζει   │
  │            από NLU     στη μνήμη            τη ροή       │
  └──────────────────────────────────────────────────────────┘
```

---

## Μέρος 1 — Entities (Οντότητες)

### Τι είναι;

Ένα entity είναι ένα δομημένο κομμάτι πληροφορίας μέσα στο μήνυμα
του χρήστη. Σκεφτείτε το σαν **υπογράμμιση** πάνω σε μια λέξη.

### Οπτικοποίηση: Εξαγωγή Entities

```
  Μήνυμα χρήστη:
  ╔══════════════════════════════════════════════════════════╗
  ║  "Θέλω να κλείσω πτήση για Θεσσαλονίκη αύριο πρωί"       ║
  ╚══════════════════════════════════════════════════════════╝
                                  │
                           NLU Pipeline
                                  │
                                  ▼
  ╔══════════════════════════════════════════════════════════╗
  ║  intent: κράτηση_πτήσης  (confidence: 0.94)              ║
  ║                                                          ║
  ║  entities:                                               ║
  ║  ┌──────────────────────────────────────────────────┐    ║
  ║  │  "Θεσσαλονίκη"                                   │    ║
  ║  │   entity: πόλη                                   │    ║
  ║  │   start: 28, end: 40                             │    ║
  ║  │   confidence: 0.97                               │    ║
  ║  └──────────────────────────────────────────────────┘    ║
  ║  ┌──────────────────────────────────────────────────┐    ║
  ║  │  "αύριο πρωί"                                    │    ║
  ║  │   entity: χρόνος                                 │    ║
  ║  │   start: 41, end: 51                             │    ║
  ║  │   confidence: 0.91                               │    ║
  ║  └──────────────────────────────────────────────────┘    ║
  ╚══════════════════════════════════════════════════════════╝
```

### Πώς ορίζονται στα Training Data

Στο αρχείο `nlu.yml`, τα entities σημειώνονται μέσα σε αγκύλες:

```yaml
- intent: κράτηση_πτήσης
  examples: |
    - Θέλω πτήση για [Θεσσαλονίκη](πόλη) [αύριο πρωί](χρόνος)
    - Κλείσε μου εισιτήριο για [Αθήνα](πόλη) [την Παρασκευή](χρόνος)
    - Πτήση προς [Ηράκλειο](πόλη)
```

Σύνταξη: `[τιμή](όνομα_entity)` — η τιμή είναι το κείμενο,
το όνομα entity είναι η κατηγορία.

### Πώς ορίζονται στο Domain

```yaml
entities:
  - πόλη
  - χρόνος
```

### Τύποι Entity Extraction

```
  ┌─────────────────────────────────────────────────────────────────┐
  │                    ΜΕΘΟΔΟΙ ΕΞΑΓΩΓΗΣ                             │
  │                                                                 │
  │  1. DIETClassifier        ML μοντέλο, μαθαίνει από παραδείγματα │
  │     ➜ Ευέλικτο, γενικεύει σε νέες τιμές                         │
  │     ➜ Π.χ. αναγνωρίζει "Λάρισα" ως πόλη ακόμα κι αν             │
  │        δεν υπήρχε στα training data                             │
  │                                                                 │
  │  2. RegexEntityExtractor  Βασισμένο σε κανονικές εκφράσεις      │
  │     ➜ Για δομημένα patterns (τηλέφωνα, emails, ΑΦΜ)             │
  │     ➜ Π.χ. regex: ^[0-9]{9}$ → entity: ΑΦΜ                      │
  │                                                                 │
  │  3. EntitySynonymMapper   Κανονικοποίηση τιμών                  │
  │     ➜ Μετατρέπει "Θεσ/νίκη", "SKG" → "Θεσσαλονίκη"              │
  │                                                                 │
  │  4. SpacyEntityExtractor  Προ-εκπαιδευμένο μοντέλο spaCy        │
  │     ➜ Αναγνωρίζει πρόσωπα, οργανισμούς, τοποθεσίες κ.λπ.        │
  │                                                                 │
  │  5. DucklingEntityExtractor  Εξωτερική υπηρεσία                 │
  │     ➜ Ημερομηνίες, ποσά, distances, temperatures                │
  │     ➜ Π.χ. "μεθαύριο στις 3" → datetime object                  │
  └─────────────────────────────────────────────────────────────────┘
```

### Roles & Groups (Ρόλοι & Ομάδες)

Όταν ένα entity εμφανίζεται πολλές φορές με διαφορετικό ρόλο:

```
  "Θέλω πτήση από Αθήνα για Θεσσαλονίκη"
                   ▲              ▲
                   │              │
            πόλη (role:         πόλη (role:
            αναχώρηση)          προορισμός)
```

Στα training data:

```yaml
- Θέλω πτήση από [Αθήνα]{"entity":"πόλη","role":"αναχώρηση"} για [Θεσσαλονίκη]{"entity":"πόλη","role":"προορισμός"}
```

## Μέρος 2 — Slots (Υποδοχές Μνήμης)

### Τι είναι;

Τα slots είναι η **μνήμη** του bot. Αποθηκεύουν πληροφορίες κατά τη
διάρκεια μιας συνομιλίας και μπορούν να **επηρεάσουν τη ροή** του διαλόγου.

### Οπτικοποίηση: Η Ζωή ενός Slot

```
  ┌─────────────────────────────────────────────────────────────┐
  │                ΚΥΚΛΟΣ ΖΩΗΣ ΕΝΟΣ SLOT                        │
  │                                                             │
  │  1. ΓΕΜΙΣΜΑ (τρόποι να πάρει τιμή):                         │
  │     ┌─────────────────────────────────────────────┐         │
  │     │  ● Αυτόματα από entity (from_entity)        │         │
  │     │    Χρήστης: "Πτήση για Αθήνα"               │         │
  │     │    → entity πόλη = "Αθήνα"                  │         │
  │     │    → slot πόλη ← "Αθήνα" (αυτόματο)         │         │
  │     │                                             │         │
  │     │  ● Από custom action (μέσω κώδικα)          │         │
  │     │    return [SlotSet("πόλη", "Αθήνα")]        │         │
  │     │                                             │         │
  │     │  ● Από trigger intent (εξωτερικό event)     │         │
  │     └─────────────────────────────────────────────┘         │
  │                          │                                  │
  │                          ▼                                  │
  │  2. ΑΠΟΘΗΚΕΥΣΗ (μνήμη συνομιλίας):                          │
  │     ┌─────────────────────────────────────────────┐         │
  │     │  tracker = {                                │         │
  │     │    slots: {                                 │         │
  │     │      "πόλη": "Αθήνα",         ← text slot   │         │
  │     │      "ημερομηνία": "2026-04-05", ← text     │         │
  │     │      "αρ_επιβατών": 2,         ← float slot │         │
  │     │      "επιβεβαίωση": true       ← bool slot  │         │
  │     │    }                                        │         │
  │     │  }                                          │         │
  │     └─────────────────────────────────────────────┘         │
  │                          │                                  │
  │                          ▼                                  │
  │  3. ΕΠΙΡΡΟΗ ΣΤΗ ΡΟΗ (ανάλογα τον τύπο slot):                │
  │     ┌─────────────────────────────────────────────┐         │
  │     │  Αν slot πόλη = κάτι → ρώτα ημερομηνία      │         │
  │     │  Αν slot πόλη = κενό → ρώτα πόλη            │         │
  │     └─────────────────────────────────────────────┘         │
  └─────────────────────────────────────────────────────────────┘
```

### Τύποι Slots

```
  ┌──────────────────────────────────────────────────────────────────────┐
  │                        ΤΥΠΟΙ SLOTS                                   │
  │                                                                      │
  │  Τύπος          Τιμή              Επιρροή στη ροή                    │
  │  ─────────────────────────────────────────────────────────────────── │
  │                                                                      │
  │  text           "Αθήνα"           Ναι — Policies ξέρουν ότι          │
  │                                    ΤΟ SLOT ΕΧΕΙ ΤΙΜΗ (αλλά           │
  │                                    δεν βλέπουν ποια τιμή)            │
  │                                                                      │
  │  bool           true/false        Ναι — Policies βλέπουν             │
  │                                    true ή false                      │
  │                                                                      │
  │  float          3.14              Ναι — Policies βλέπουν             │
  │                                    αν είναι πάνω/κάτω από            │
  │                                    min_value και max_value           │
  │                                                                      │
  │  categorical    "economy"         Ναι — Policies βλέπουν             │
  │                 "business"         ποια κατηγορία είναι              │
  │                                                                      │
  │  list           ["Αθήνα",         Ναι — Policies ξέρουν              │
  │                  "Θεσ/νίκη"]       αν η λίστα είναι κενή ή όχι       │
  │                                                                      │
  │  any            οτιδήποτε         ΟΧΙ — αποθηκεύει τιμή              │
  │                                    αλλά ΔΕΝ επηρεάζει τη ροή         │
  └──────────────────────────────────────────────────────────────────────┘
```

### Πώς ορίζονται στο Domain

```yaml
entities:
  - πόλη
  - χρόνος
  - αρ_επιβατών

slots:
  πόλη:
    type: text
    influence_conversation: true
    mappings:
      - type: from_entity
        entity: πόλη

  ημερομηνία:
    type: text
    influence_conversation: true
    mappings:
      - type: from_entity
        entity: χρόνος

  αρ_επιβατών:
    type: float
    min_value: 1
    max_value: 9
    influence_conversation: false
    mappings:
      - type: from_entity
        entity: αρ_επιβατών

  επιβεβαίωση:
    type: bool
    influence_conversation: true
    mappings:
      - type: from_intent
        intent: affirm
        value: true
      - type: from_intent
        intent: deny
        value: false
```

### Slot Mappings (Τρόποι γεμίσματος)

```
  ┌──────────────────────────────────────────────────────────────────────┐
  │                     SLOT MAPPINGS                                    │
  │                                                                      │
  │  Τύπος               Πώς γεμίζει              Παράδειγμα             │
  │  ──────────────────────────────────────────────────────────────────  │
  │                                                                      │
  │  from_entity         Αυτόματα, όταν           Χρήστης λέει "Αθήνα"   │
  │                      βρεθεί entity στο         → slot πόλη = "Αθήνα" │
  │                      μήνυμα                                          │
  │                                                                      │
  │  from_text           Ολόκληρο το μήνυμα       Χρήστης λέει κάτι      │
  │                      αποθηκεύεται             → slot = πλήρες text   │
  │                                                                      │
  │  from_intent         Σταθερή τιμή αν          intent=affirm          │
  │                      ανιχνευθεί intent        → slot = true          │
  │                                                                      │
  │  from_trigger_intent Σταθερή τιμή από         Εξωτερικό trigger      │
  │                      εξωτερικό event          → slot = τιμή          │
  │                                                                      │
  │  custom              Μέσω custom action       Python κώδικας         │
  │                      (Python)                 SlotSet(...)           │
  └──────────────────────────────────────────────────────────────────────┘
```

## Μέρος 3 — Πώς Συνδέονται Entity ↔ Slot ↔ Dialogue

### Πλήρης ροή μιας συνομιλίας

```
  Χρήστης: "Θέλω πτήση για Θεσσαλονίκη αύριο"
  ═══════════════════════════════════════════════════════════════

  ΒΗΜΑ 1 — NLU Pipeline (εξαγωγή)
  ┌───────────────────────────────────────────────┐
  │  intent: κράτηση_πτήσης                       │
  │  entities:                                    │
  │    πόλη = "Θεσσαλονίκη"                       │
  │    χρόνος = "αύριο"                           │
  └──────────────────────┬────────────────────────┘
                         │
                         ▼
  ΒΗΜΑ 2 — Slot Filling (αποθήκευση)
  ┌───────────────────────────────────────────────┐
  │  tracker.slots:                               │
  │    πόλη ← "Θεσσαλονίκη"   (from_entity)       │
  │    ημερομηνία ← "αύριο"    (from_entity)      │
  │    αρ_επιβατών ← None      (δεν αναφέρθηκε)   │
  │    επιβεβαίωση ← None      (δεν αναφέρθηκε)   │
  └──────────────────────┬────────────────────────┘
                         │
                         ▼
  ΒΗΜΑ 3 — Policies (απόφαση)
  ┌───────────────────────────────────────────────┐
  │  Οι policies βλέπουν:                         │
  │    • intent = κράτηση_πτήσης                  │
  │    • slot πόλη = HAS_VALUE ✓                  │
  │    • slot ημερομηνία = HAS_VALUE ✓            │
  │    • slot αρ_επιβατών = EMPTY ✗               │
  │                                               │
  │  Απόφαση: ρώτα για αριθμό επιβατών            │
  └──────────────────────┬────────────────────────┘
                         │
                         ▼
  ΒΗΜΑ 4 — Απάντηση Bot
  ┌───────────────────────────────────────────────┐
  │  Bot: "Πόσοι επιβάτες θα ταξιδέψουν;"         │
  └───────────────────────────────────────────────┘

  ═══════════════════════════════════════════════════════════════

  Χρήστης: "2 άτομα"

  ┌───────────────────────────────────────────────┐
  │  NLU → entity: αρ_επιβατών = 2                │
  │  Slot filling → αρ_επιβατών ← 2               │
  │  Policies → ΟΛΑ τα slots γεμάτα!              │
  │  Απόφαση → ζήτα επιβεβαίωση                   │
  └───────────────────────────────────────────────┘

  Bot: "Πτήση για Θεσσαλονίκη, αύριο, 2 επιβάτες. Σωστά;"

  ═══════════════════════════════════════════════════════════════

  Χρήστης: "Ναι"

  ┌───────────────────────────────────────────────┐
  │  NLU → intent: affirm                         │
  │  Slot filling → επιβεβαίωση ← true            │
  │  Policies → εκτέλεσε κράτηση                  │
  └───────────────────────────────────────────────┘

  Bot: "Η κράτησή σας ολοκληρώθηκε! ✈"
```

### Σημαντική διάκριση: influence_conversation

```
  ┌──────────────────────────────────────────────────────────────────┐
  │              influence_conversation: true vs false               │
  │                                                                  │
  │  TRUE:  Οι policies ΒΛΕΠΟΥΝ αν το slot έχει τιμή.                │
  │         Η τιμή του slot αλλάζει τη συμπεριφορά του bot.          │
  │         ➜ "Αν ξέρω την πόλη, ρώτα ημερομηνία"                    │
  │                                                                  │
  │  FALSE: Οι policies ΔΕΝ βλέπουν το slot.                         │
  │         Η τιμή αποθηκεύεται αλλά δεν επηρεάζει τη ροή.           │
  │         ➜ Χρήσιμο για αποθήκευση δεδομένων που χρειάζονται       │
  │            μόνο σε custom actions (π.χ. API responses)           │
  └──────────────────────────────────────────────────────────────────┘
```

#### Παράδειγμα: Πώς το influence_conversation αλλάζει τη ροή

Domain — ορίζουμε ένα slot με `influence_conversation: true`:

```yaml
slots:
  πόλη:
    type: text
    influence_conversation: true
    mappings:
      - type: from_entity
        entity: πόλη
```

Stories — γράφουμε **δύο ξεχωριστά stories** για κάθε περίπτωση.
Οι policies μαθαίνουν ότι το ίδιο intent οδηγεί σε διαφορετική
ενέργεια ανάλογα με το αν το slot έχει τιμή ή όχι:

```yaml
stories:
# Ο χρήστης ανέφερε πόλη → slot γεμάτο → ρώτα ημερομηνία
- story: παροχή πόλης
  steps:
    - intent: κράτηση_πτήσης
    - slot_was_set:
        - πόλη: "Θεσσαλονίκη"
    - action: utter_ask_date

# Ο χρήστης ΔΕΝ ανέφερε πόλη → slot κενό → ρώτα πόλη
- story: χωρίς πόλη
  steps:
    - intent: κράτηση_πτήσης
    - action: utter_ask_city
```

Αν το slot είχε `influence_conversation: false`, οι policies δεν θα
έβλεπαν καμία διαφορά μεταξύ των δύο περιπτώσεων και δεν θα μπορούσαν
να διαλέξουν διαφορετική ενέργεια.

## Μέρος 4 — Forms (Φόρμες)

Τα Forms είναι ο πιο κομψός τρόπος να μαζέψετε entities σε slots.
Ορίζετε ποια slots χρειάζεστε και το Rasa ρωτάει αυτόματα για τα κενά.

```
  ┌──────────────────────────────────────────────────────────────────┐
  │                    FORM: κράτηση_πτήσης_form                     │
  │                                                                  │
  │  Απαιτούμενα slots:                                              │
  │    ┌──────────────┬───────────┬─────────────────────────┐        │
  │    │ Slot         │ Γεμάτο;   │ Ερώτηση αν κενό         │        │
  │    ├──────────────┼───────────┼─────────────────────────┤        │
  │    │ πόλη         │  ✓ Ναι    │ —                       │        │
  │    │ ημερομηνία   │  ✗ Όχι    │ "Πότε θέλετε να πετάξ;" │        │
  │    │ αρ_επιβατών  │  ✗ Όχι    │ "Πόσοι επιβάτες;"       │        │
  │    └──────────────┴───────────┴─────────────────────────┘        │
  │                                                                  │
  │  Το Rasa ρωτάει ΑΥΤΟΜΑΤΑ μόνο για τα κενά slots,                 │
  │  μέχρι να συμπληρωθούν όλα → μετά τρέχει submit action.          │
  └──────────────────────────────────────────────────────────────────┘
```

Ορισμός στο domain:

```yaml
forms:
  κράτηση_πτήσης_form:
    required_slots:
      - πόλη
      - ημερομηνία
      - αρ_επιβατών
```

## Συνοπτικός Πίνακας

```
  ┌────────────┬──────────────────────┬───────────────────────────────┐
  │ Έννοια     │ Πού ζει              │ Τι κάνει                      │
  ├────────────┼──────────────────────┼───────────────────────────────┤
  │ Entity     │ NLU pipeline         │ Εξάγει πληροφορία από         │
  │            │ (nlu.yml, domain)    │ το μήνυμα του χρήστη          │
  ├────────────┼──────────────────────┼───────────────────────────────┤
  │ Slot       │ Dialogue tracker     │ Αποθηκεύει τιμή στη μνήμη     │
  │            │ (domain.yml)         │ και (προαιρετικά) επηρεάζει   │
  │            │                      │ τη ροή του διαλόγου           │
  ├────────────┼──────────────────────┼───────────────────────────────┤
  │ Mapping    │ domain.yml           │ Ορίζει ΠΩΣ γεμίζει ένα slot   │
  │            │ (slots.mappings)     │ (from_entity, from_intent…)   │
  ├────────────┼──────────────────────┼───────────────────────────────┤
  │ Form       │ domain.yml + rules   │ Αυτόματη συλλογή πολλαπλών    │
  │            │                      │ slots μέσω ερωτήσεων          │
  └────────────┴──────────────────────┴───────────────────────────────┘
```
